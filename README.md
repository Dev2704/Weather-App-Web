🌤️ Weather App

A clean and responsive weather application built with HTML, CSS, and Vanilla JavaScript that fetches real-time weather information using the wttr.in Weather API.

This project was built to strengthen my understanding of asynchronous JavaScript, API integration, DOM manipulation, and error handling before moving on to frontend frameworks like React.

⸻

✨ Features

* 🔍 Search weather by city name
* 🌡️ Current temperature
* 🥵 Feels like temperature
* 💧 Humidity
* ☁️ Cloud cover
* 🌬️ Wind speed
* ⌨️ Search using the Enter key
* ⏳ Loading indicator while fetching data
* ❌ Error handling for invalid city names or failed requests
* 🎨 Clean and responsive UI

⸻

🛠️ Built With

* HTML5
* CSS3
* Vanilla JavaScript (ES6+)
* Fetch API
* Async / Await
* wttr.in Weather API

🚀 What I Learned

Building this project helped me understand how frontend applications communicate with external APIs and display real-time data to users.

Some of the key concepts I practiced include:

* Working with asynchronous JavaScript using async / await
* Fetching and parsing JSON data from an API
* Handling API failures gracefully using try...catch
* Updating the DOM dynamically based on fetched data
* Structuring code into reusable functions
* Creating a better user experience with loading and error states

⸻

🤖 About AI

AI was used throughout this project as a learning assistant, not as a replacement for programming.

I used it to:

* Understand JavaScript concepts
* Debug issues
* Review code structure
* Explore different implementation approaches
* Learn best practices instead of copying complete solutions

The application’s logic, debugging process, and implementation were developed through an iterative learning process where I wrote, tested, modified, and improved the code myself.

I believe AI is most valuable when it helps developers learn faster, think more clearly, and build better software—not when it replaces understanding.
